Trello API (v1): Setting Up Authentication

Securely interact with Trello by using an API key and token. This document covers the prerequisites, step-by-step instructions, and common errors you might encounter.

Prerequisites

  • A valid Trello account (sign up at Trello.com )
  • Familiarity with basic REST API concepts
  • A tool to send HTTP requests (e.g., Postman or curl)

Authentication Steps

  1. Generate an API Key: Visit the Trello API Key Page
  2. Generate an Access Token: Follow the instructions on the API Key page to obtain a token that grants your application the necessary permissions.

Example Requests

Below are sample requests in two common formats. Replace YOUR_API_KEY and YOUR_ACCESS_TOKEN with your own credentials.

cURL
Node.js
curl --request GET \
  --url "https://api.trello.com/1/members/me?key=YOUR_API_KEY&token=YOUR_ACCESS_TOKEN" \
  --header "Accept: application/json"
        
// Using fetch (Node 18+ or via node-fetch)
fetch("https://api.trello.com/1/members/me?key=YOUR_API_KEY&token=YOUR_ACCESS_TOKEN", {
  method: "GET",
  headers: {
    "Accept": "application/json"
  }
})
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

Common Errors

Error Code Possible Cause Solution
401 Unauthorized Invalid or missing API Key / Token Double-check your API key and token on the Trello API Page .
404 Not Found Incorrect endpoint or resource ID Verify you are using the correct URL and resource identifiers.
429 Too Many Requests Rate limit exceeded Wait before sending more requests. Consult Trello’s rate limit guidelines .
